Decrypts the data using Cipher Block Chain (CBC) AES implemented directly on the Intel Chip.

Decrypts the passed input data using CBC. The function is implemented using intel Intrinsics for greater performance.

Note
CBC requires padding, which this function assumes you will remove yourself after returning
Warning
The input length must be a multiple of 16. No checks are run to ensure input, ivec, or epoch key are the correct lengths

Possible values for the key mode and what it specifies:

  • aes_128:
    AES-128 [key = 128bits, AES rounds = 10]
  • aes_192:
    AES-192 [key = 192bits, AES rounds = 12]
  • aes_256:
    AES-256 [key = 256bits, AES rounds = 14]
Parameters
inptThe data to decrypt using AES and CBC
outtA pointer to a malloced location where the decrypted data will be written
ivecThe IV (Initial Vector) to be used for CBC decryption
clengthThe length of the input cipher [in bytes] which is also the output (message) length
epoch_keyThe key (passed by the user) that will be used for the key expansion to make the key schedule to decrypt
keymodeThe AES mode (also defines the key length and number of rounds) [see the possible values above]

https://en.wikipedia.org/wiki/Block_cipher_mode_of_operation#Cipher_Block_Chaining_(CBC)

Encrypts or Decrypts the data using Counter Mode (CTR) AES implemented directly on the Intel Chip.

Encrypts or Decrypts the passed input data using CTR. The function is implemented using intel Intrinsics for greater performance.

Note
Due to the nature of CTR, encryption and decryption are the same so that does not have to be specified. If a non encrypted message is passed it will be encrypted, if an encrypted message is passed it will be decrypted
Warning
No checks are run to ensure input, ivec, or epoch key are the correct lengths

Possible values for the key mode and what it specifies:

  • aes_128:
    AES-128 [key = 128bits, AES rounds = 10]
  • aes_192:
    AES-192 [key = 192bits, AES rounds = 12]
  • aes_256:
    AES-256 [key = 256bits, AES rounds = 14]
Parameters
inptThe data to decrypt/decrypt using AES and CBC
outtA pointer to a malloced location where the decrypted/encrypted data will be written
ivecThe IV (Initial Vector) to be used during the CTR process
mlengthThe length of the input [in bytes] which is also the output length
epoch_keyThe key (passed by the user) that will be used for the key expansion to make the key schedule to encrypt/decrypt
keymodeThe AES mode (also defines the key length and number of rounds) [see the possible values above]

Decrypts the data using AES implemented directly on the Intel Chip.

Decrypts the data passed with the specified Key Schedule through AES using the key length set. The function is implemented using intel Intrinsics for greater performance.

Warning
The decryption is done directly on the passed data array which must be 128 bits (16 bytes)
uint8_t * fullCipher = ...;
uint8_t * userKey = ...; // 128bits using AES-128
__m128i * keySchedule = loadKeyExpansion(userKey, aes_128);
// Decrypt the first 16 bytes AES-128
aes_ni_dec((__m128i *)fullCipher[0], keySchedule, aes_128);
Parameters
dataThe data to decrypt
key_scheduleThe key schedule to use
keymodeThe key mode specifying the key schedule length and AES mode
